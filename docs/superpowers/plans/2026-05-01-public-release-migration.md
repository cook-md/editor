# Public Release Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate Cook Editor from `cook-md/editor-private` to public `cook-md/editor`, restructure for public consumption, and relocate working directory from `Cooklang/editor/` to `Cooklang/editor-release/`.

**Architecture:** Restructure in place in the private working directory through a series of small commits, then switch the git remote and force-push to the public repo, then rename the local directory. Each restructuring change is its own commit so the public history reads cleanly.

**Tech Stack:** Theia 1.70 monorepo, Lerna 9, npm workspaces, electron-builder, GitHub Actions for releases.

## Rollback procedure

If something goes catastrophically wrong before Phase 2 (Task 13+) is complete:

```bash
git reset --hard private/main  # if remote rename has happened
# or
git reset --hard origin/main   # before remote rename
```

If something goes wrong after the force-push (Task 14):

The `private` remote (created in Task 13) still has the original private repo state. Force-push from there back to public:

```bash
git fetch private
git push --force origin private/main:main
```

This restores the public repo to its pre-Task-14 state.
