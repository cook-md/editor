# Public Release Migration

## Goal

Migrate the Cook Editor codebase from the private repo (`cook-md/editor-private`)
to public (`cook-md/editor`), restructure for public consumption, and relocate
the working directory from `Cooklang/editor/` to `Cooklang/editor-release/`.

## Decisions

| Decision | Choice |
|---|---|
| Git history strategy | **Preserve full history** — push private repo's history as-is |
| Restructuring scope | **Promote app out of `examples/`** and delete unused examples |
| New app path | `app/` (singular, minimal) |
| Theia upstream artifact cleanup | Delete `sample-plugins/`, `devfile.yaml`; rewrite `CHANGELOG.md`, `README.md`, `CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`, `SECURITY.md` |
| Existing public repo's 4 commits | Force-push and discard (their content already exists in private repo at v0.1.0-alpha.9) |
| Sensitive content audit | Skip (trust past hygiene) |
| npm package name | Rename `@theia/example-electron` → `@cook-md/editor-app` |

## Non-goals

- No git history rewrite or secrets scrubbing
- No renaming of upstream Theia packages (`@theia/*` framework dependencies must stay)
- No new features or refactoring beyond what's required for the rename
- No migration of `~/.claude/projects/` memory (user handles separately if needed)

## Phase 1: Restructure (in current `editor/` working directory)

Each numbered item is its own commit so history reads cleanly.

### 1.1 Move `examples/electron` → `app/`

```
git mv examples/electron app
```

Then update path references in the same commit:

- Root `package.json` workspaces glob (`examples/electron` → `app`)
- `lerna.json` `packages` glob
- `.github/workflows/release.yml` — paths, `working-directory:`, artifact paths
- `tsconfig.json` references (project references pointing at `examples/electron`)
- `CLAUDE.md` mentions of `examples/electron/`
- `doc/*.md` mentions
- Any `scripts/*.js|sh` referencing the path

**Verification:** `grep -r "examples/electron" .` (excluding `node_modules`,
`package-lock.json`) returns no matches.

### 1.2 Rename npm package `@theia/example-electron` → `@cook-md/editor-app`

References to update (3 files identified):

- `app/package.json` (the `name` field, was `examples/electron/package.json`)
- `package-lock.json` — regenerated by `npm install`
- `doc/Migration.md` — replace mention or delete the line

**Verification:** `grep -r "@theia/example-electron" .` returns no matches
(except in `package-lock.json` which is regenerated).

### 1.3 Delete unused examples

```
rm -rf examples/
```

Removes: `browser/`, `browser-only/`, `api-samples/`, `api-tests/`,
`playwright/`, `ovsx-router-config.json`. (`electron/` is already moved.)

### 1.4 Delete `sample-plugins/`

```
rm -rf sample-plugins/
```

Theia upstream's sample namespace plugin demo. Unused.

### 1.5 Delete `devfile.yaml`

Eclipse Che dev-environment config. Irrelevant to a desktop app.

### 1.6 Rewrite `CHANGELOG.md`

Replace Theia upstream framework changelog (2017+) with Cook Editor's own
history: `v0.1.0-alpha.1` through `v0.1.0-alpha.9`. Source the entries from
existing release tags / commit messages (`git log --oneline --tags`).

### 1.7 Rewrite `README.md`

Drop Theia upstream content. New structure:

- One-paragraph "What is Cook Editor" intro
- Hero screenshot
- Download table (keep existing — already public-ready)
- Linux AppImage / sandbox notes (keep existing)
- "Build from source" → short pointer to `doc/Developing.md`
- License & attribution (EPL-2.0, Theia upstream, NOTICE.md pointer)

Don't bury the download links — visitors come for binaries first.

### 1.8 Rewrite `CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`, `SECURITY.md`

Cook Editor-flavored. `SECURITY.md` must point to a real disclosure address
(user to confirm: `dubovskoy.a@gmail.com` or a `security@cooklang.org` alias).

### 1.9 CLAUDE.md sanity pass

CLAUDE.md is fine in public repos generally. Surface its contents to the user
during implementation so they can decide:

- Keep as-is (developer-facing, not sensitive)
- Slim down (drop merge notes, internal package mentions)
- Move to `.claude/CLAUDE.md` (still works, less prominent)

### 1.10 Build verification

Before pushing:

```
npm install
cd app && npm run bundle
cd .. && npm run start:electron
```

Verify the app launches, opens a recipe, syntax highlighting works, autocomplete
works. Don't push if anything is broken.

## Phase 2: Switch remote and push

1. `git remote rename origin private` (preserve old private remote as `private`)
2. `git remote add origin git@github.com:cook-md/editor.git`
3. `git push --force origin main` — overwrites the 4 throwaway commits in
   the public repo
4. `git push --tags origin` — ships `v0.1.0-alpha.1` through `v0.1.0-alpha.9`
   tags. **Critical** for the Releases page on GitHub.

## Phase 3: Move directory locally

1. Verify `../editor-release/` is just the placeholder (4 commits, licenses +
   README — content already in current repo)
2. `rm -rf ../editor-release`
3. `mv ../editor ../editor-release`
4. Heads-up to user (manual): IDE workspaces / shell aliases / Claude Code
   project paths pointing at `Cooklang/editor` need updating

## Phase 4: GitHub-side cleanup (manual, user does it)

- Repo description, topics, website link → cooklang.org
- Verify Releases page carries the alpha tags
- Branch protection on `main` (user's choice on rules)
- Disable Wiki, Discussions if not wanted
- Update social-preview image if any

## Risks

| Risk | Mitigation |
|---|---|
| `git mv examples/electron app` breaks the build (path refs in webpack/lerna/tsconfig) | Phase 1.10 build verification — must pass before any push |
| Force-pushing breaks anyone who cloned the public skeleton | Public repo is brand-new, ~no clones. Acceptable. |
| Tags don't push or release artifacts get unlinked | Phase 2 step 4 explicitly pushes tags. Verify after push by checking GitHub Releases page. |
| Renaming npm package breaks internal `require()` calls | Only 3 files reference it (verified via grep). Update all in single commit. |
| Sensitive content in history despite skipping audit | User accepted risk (decision B in Q6) |
